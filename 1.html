網頁程式設計教學

第一章：網頁設計基本概念

1-1 網站基本概念

本節介紹網站、網頁、全球資訊網(WWW)的基本定義，以及網站的規劃、設計流程和運作方式。

  • 全球資訊網 (WWW)：基於超文本(Hypertext)技術，透過HTTP協定在網際網路上提供多媒體資訊服務的系統。
  • 網站 (Website)：由多個相關網頁組成的集合，通常有一個主要的入口頁面，稱為首頁 (Homepage)。
  • 網頁 (Web Page)：構成網站的基本單位，用來展示文字、圖片、影音等資訊。
  • 網站設計流程：
    1. 內容規劃：確定目標、架構。
    2. 設計與製作：視覺設計、HTML/CSS編寫。
    3. 測試與發行：功能測試、上傳伺服器。
    4. 流量分析：監控與優化。
  • 網頁運作原則：使用者透過瀏覽器請求 -> 伺服器回應HTML文件 -> 瀏覽器解析並顯示網頁。
  • 響應式網頁設計 (RWD)：讓網頁能在不同尺寸的裝置（電腦、平板、手機）上都有良好的瀏覽體驗。
  • 一頁式網站：將所有內容放在單一頁面，透過滾動瀏覽，適合內容較少或特定行銷目的的網站。

1-2 網頁設計程式語言

網頁的構成涉及多種語言和技術，主要分為瀏覽器端和伺服器端。

瀏覽器端 (Client-Side)

  • HTML (HyperText Markup Language)：定義網頁的結構和內容。
  • CSS (Cascading Style Sheets)：描述網頁的外觀和樣式。
  • JavaScript：實現網頁的互動效果和動態功能。(DHTML是早期結合HTML, CSS, JavaScript的技術概念)
  • XML (Extensible Markup Language)：用於描述和傳輸資料，與HTML不同，它著重於資料本身而非呈現。

伺服器端 (Server-Side)

這些語言在伺服器上執行，處理業務邏輯、資料庫互動等，然後生成HTML傳送給瀏覽器。

  • PHP
  • ASP.NET
  • JSP (JavaServer Pages)
  • Python (搭配 Django/Flask 框架)
  • Ruby (搭配 Rails 框架)
  • Node.js (使用 JavaScript)
  • CGI (Common Gateway Interface)：一種較早期的伺服器與外部程式溝通的標準。

MVC 架構

一種常見的軟體設計模式，將應用程式分為三層：

  • Model (模型)：處理資料和業務邏輯。
  • View (視圖)：負責使用者介面的呈現 (通常是 HTML/CSS/JS)。
  • Controller (控制器)：接收使用者請求，調度模型和視圖。

Web API (應用程式介面)

讓不同的應用程式可以互相溝通、交換資料或服務的介面。例如，在地圖上顯示店家位置（使用 Google Maps API）、使用社群帳號登入等。

第二章：HTML 基本概念

2-1 認識 HTML

HTML 是構成網頁的基礎。HTML5 是目前的最新標準，它簡化了語法，增加了許多新功能（如影音支援、繪圖等），並提升了語意化。

可以使用 W3C 的驗證工具 (http://validator.w3.org) 來檢查 HTML 程式碼是否符合標準。

2-2 HTML 檔案命名的原則

  • 副檔名：使用 .html 或 .htm。
  • 字元：建議使用半形小寫英文字母、數字、底線 _ 或連字號 -。避免使用中文、全形字元或空格。
  • 首頁檔名：通常命名為 index.html (或伺服器指定的名稱如 default.htm)。
  • 資料夾管理：將網站所有檔案放在一個根資料夾，並建立子資料夾 (如 images, css, js) 來組織檔案。

檔案路徑

  • 絕對路徑：完整的 URL，如 https://www.example.com/images/logo.png。
  • 相對路徑：相對於目前 HTML 檔案的路徑。
    • images/logo.png：目前資料夾下的 images 資料夾內的 logo.png。
    • ../styles/main.css：上一層資料夾下的 styles 資料夾內的 main.css。
  • 根目錄相對路徑：從網站根目錄開始的路徑，以 / 開頭，如 /images/logo.png。
